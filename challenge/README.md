# Fix My Code Challenge

## Background Context

**Fix my code** is a new type of project where the goal is to dive into an existing, broken codebase and fix its issues.

In this project, the challenge is twofold: sometimes the programming language will be familiar, and sometimes it won't. The objective is not to rewrite the entire application or recode everything from scratch, but to analyze, debug, and surgically fix the existing bugs.

The initial files for all solutions are retrieved from the [Fix_My_Code_Challenge](https://github.com/) repository.

---

## Requirements

### General

- **Allowed Editors:** `vi`, `vim`, `emacs`
- **Compilation Environment:** All files will be compiled/tested on **Ubuntu 20.04 LTS**
- **Formatting:** All files must end with a new line.
- **Documentation:** A `README.md` file at the root of the project folder is mandatory.

---

## Technical Approach & Methodology

1.  **Analyze:** Run the broken code and inspect the error messages or unexpected outputs.
2.  **Isolate:** Locate the specific function or logic causing the bug without changing the overall architecture.
3.  **Fix:** Apply the minimal necessary correction to make the code compliant with the expected behavior.
4.  **Validate:** Ensure that the solution compiles and runs correctly on Ubuntu 20.04 LTS and that all edge cases are handled.
